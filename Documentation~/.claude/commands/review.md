# 代码审查

对指定的代码文件或目录进行全面的代码审查。

## 参数
- $TARGET: 要审查的文件或目录路径

## 审查维度

### 1. 代码质量
- 命名规范（类名、方法名、变量名）
- 代码格式和缩进
- 注释的完整性和准确性
- 代码复杂度（方法长度、嵌套层级）

### 2. 架构设计
- 是否遵循框架的模块化设计
- 依赖关系是否合理
- 是否符合 SOLID 原则
- 是否正确使用设计模式

### 3. 性能优化
- 内存分配（GC 压力）
- 循环和迭代的效率
- 缓存策略
- 异步操作的正确使用

### 4. 安全性
- 输入验证
- 资源访问控制
- 异常处理
- 线程安全

### 5. Unity 特定
- MonoBehaviour 的正确使用
- 协程和 UniTask 的正确使用
- 资源管理（加载、卸载、引用计数）
- 序列化和 Inspector 配置

### 6. 框架规范
- 是否遵循 Moirai Framework 的编码规范
- 是否正确使用框架提供的工具和组件
- 是否符合框架的事件系统使用规范
- 是否正确处理模块生命周期

## 审查流程

1. **静态分析**：检查代码结构和语法
2. **逻辑分析**：检查业务逻辑的正确性
3. **性能分析**：识别潜在的性能问题
4. **安全分析**：检查安全漏洞
5. **规范检查**：检查是否符合编码规范
